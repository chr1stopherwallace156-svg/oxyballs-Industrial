# Vehicle Configurations (Ephemeral)

Under Hard Rule 0, a vehicle configuration is a **thin assembly pointer** into the component graph.

| Config ID | File |
|---|---|
| `F450_2019_REG_CAB_145_3_WB_4X2` | [VEH_CFG_F450_2019_REG_CAB_145_3.json](VEH_CFG_F450_2019_REG_CAB_145_3.json) |

Future platforms (E-450, Transit) add new config files that **reuse** component passports where interfaces match — they do not fork monolithic vehicle meshes.
