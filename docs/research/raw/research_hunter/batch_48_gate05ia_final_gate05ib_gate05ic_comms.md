Gate 05I-A — Low-Voltage Driver Safety Logic Verification (Final Baseline)
This gate defines the automated and manual verification protocol for core driver-safety functions. All operations are confined exclusively to a production-like low-voltage bench integration environment.
To prevent physical hazards, structural damage, or invalid safety claims, this verification is governed by three absolute constraints:
CRITICAL_RESTRICTION: Zero vehicle motion. Wheels-on-ground operation, chassis dynamometer testing, and real Ford factory bus transmission (CAN_1) are completely forbidden.
CRITICAL_RESTRICTION: Zero live high-voltage testing. All supplier controllers (Inverter and BMS logic boards) must run on low-voltage auxiliary power (12V nominal). High-voltage rails must remain completely unpowered.
CRITICAL_RESTRICTION: Zero road-test approval. This protocol validates low-voltage software and hardware logic paths only; it does not approve physical vehicle driving or driver operation.
1. Hardwired Fault & Signal Boundary Rules
CAN_1 Fault Injection Constraint: CAN_1 fault injection is permitted only on a protected bench harness or simulated OEM network. Direct short-circuiting of the production chassis interface pins is strictly prohibited.
Hardwired Interlock Interruption: The emergency stop mechanism removes the low-voltage control path through the approved hardwired safety loop. Coil voltage decay, relay drop-out time, and output behavior must be measured and compared against the verified schematic and component datasheets to account for physical induction and mechanical lag.

3. Service Clear Precondition and Lockout Rules
Service Clear Operational Law: A UDS Diagnostic Session command (e.g., Service 0x14) may clear software diagnostic records only. It must not clear active hardwired faults, active HVIL faults, active E-stop, BMS no-discharge, isolation faults, or any unresolved safety latch.
Precondition Requirements for Diagnostic Recovery:
No Active Fault Input: Physical fault signals (such as open HVIL, active E-stop loop ground path, or out-of-range APPS tracking) must be resolved at the hardware layer prior to sending the command.
Zero Torque Command: The VCU internal torque model must output zero active request bytes.
Charger Disconnected: Proximity and pilot signal states must verify the physical charging interface is decoupled.
Safe/Neutral State: The VCU shift-state register must be in Neutral.
Technician Authorization: Secured access levels (e.g., UDS Security Access Service 0x27) must validate diagnostic clearance privileges.
Fault Source Reviewed: Software registers must verify that latched non-volatile DTCs have been analyzed and verified as clearable.

Gate 05I-C — Low-Voltage Communications Integration
With safety logic and physical interlocks validated, the next sequence on the bench is the low-voltage network interface layer. This step must prove network bus integrity, frame filtering, and database synchronization without ever exposing a live network wire to CAN_1 leakage.
Core Verification Requirements:
CAN_2 Bus Performance: VCU <-> inverter logic board data tracking, parsing, and latency verification.
CAN_3 Bus Performance: VCU <-> BMS/PDU logic board safety parameter frame exchange.
Display Node Communication: Serial or CAN display node packet routing, frame dropouts, and error warning screens.
Diagnostic Tool Communication: Secure UDS session handling, parameter identification (PID) reading, and diagnostics routing.
System Heartbeat Behavior: Loss-of-comm timeout limits (e.g., VCU torque command timeout drops to zero if inverter heartbeat fails for >100 ms).
Message Filtering & Rejection: Verify wrong-DTC, wrong-ID, or corrupted DBC packet rejection.
DBC Version Matching: Verify matching message structures, scaling, offsets, and bit positions across all nodes.
Bus Load Integrity: Monitor network health and error frame rates under simulated peak bus loading (>75% utilization).
Strict CAN_1 Isolation: Zero frame leakages, ACK responses, or active transmissions onto CAN_1 during communications exercises.
