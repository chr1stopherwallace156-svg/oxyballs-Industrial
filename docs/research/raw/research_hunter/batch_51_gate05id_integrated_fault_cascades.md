Gate 05I-C Cleanups & Technical Refinements
Before transitioning to Gate 05I-D, the following technical corrections are applied to Gate 05I-C to maintain rigorous engineering standards:
1. Unified Sleep-Current Targets
The sleep-current targets are defined as follows. Final parameters will be finalized pending engineering review, datasheet audits, and parasitic drain budget allocation:

BMS_logic_sleep_current_target = BENCH_TARGET_PROFILE
PDU_logic_sleep_current_target = BENCH_TARGET_PROFILE
Inverter_logic_sleep_current_target = BENCH_TARGET_PROFILE
Display_sleep_current_target = BENCH_TARGET_PROFILE
Total_system_sleep_current_target <= 4.0 mA (Initial overall bench target)
2. Protocol vs. Application-Layer Fault Testing
Fault injection is strictly separated by network layers:
Physical / Protocol-Layer Faults: Bad CRC, bit-stuffing, form, and bit errors are protocol-layer faults. These must be injected with a specialized hardware CAN fault-injection tool (FIU) capable of corrupting frames below the normal application layer.
Application-Layer Faults: Wrong ID, wrong source address, wrong DLC, bad alive counter, bad application checksum, and out-of-range decoded values are evaluated as application-layer validation tests inside the MCU software parser.
3. Brownout NVM Safeguards
To guarantee parameter storage integrity during power loss:
NVM Preservation Rule: The VCU shall preserve NVM integrity during brownout events. If a graceful save is required, the hardware early-warning threshold, backup hold-up capacitance, write-time budget, and flash memory endurance must be verified.
Initial Bench Profile: 6.0V drop threshold / 9.0V recovery threshold. Final values are pending VCU hardware design and physical bench validation.

Gate 05I-D Verification Exit Criteria
The system cannot pass Gate 05I-D and proceed to physical vehicle integration unless all the following criteria are met:
Zero CAN_1 Leakage: Across all 10 integrated fault cascades, the CAN_1 physical transceiver TXD pin must be logged continuously at high speed. Any transition of the VCU TXD pin to dominant (low) or any active frame transmission/acknowledgment triggers an immediate Gate Failure.
No Safe-State Overrun: The maximum time allowed between any safety trigger event and the software torque command reaching 0 Nm must be less than or equal to 100 ms (with 50 ms target for critical physical interlocks).
Non-Volatile Memory Integrity: Active fault latches must survive simulated brownouts and unexpected power cycles, ensuring that a driver cannot reset a critical safety-latching error simply by cycling the low-voltage ignition key.
Once signed off, the low-voltage bench assembly is certified safe for installation into the physical vehicle chassis, initiating the physical commissioning phases.
